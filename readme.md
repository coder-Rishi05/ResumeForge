## рџљЂ Getting Started

### Prerequisites

- Node.js v18+
- A [Groq](https://console.groq.com/) API key (free tier available)
- A [Brevo](https://www.brevo.com/) account and API key (300 emails/day free)

### 1. Clone the repository

```bash
git clone https://github.com/your-username/resumeforge.git
cd resumeforge
```

### 2. Backend Setup

```bash
cd resume-backend
npm install
```

Create a `.env` file based on `.env.example`:

```env
PORT=5000
GROQ_API_KEY=your_groq_api_key_here
BREVO_API_KEY=your_brevo_api_key_here
EMAIL_USER=your_verified_sender_email@example.com
```

Start the backend:

```bash
npm run dev
```

### 3. Frontend Setup

```bash
cd resume-frontend
npm install
npm run dev
```

The frontend runs on `http://localhost:5173` and the backend on `http://localhost:5000`.

## рџ”Њ API Reference

### `POST /api/resume/analyse`

Analyse a resume PDF against a target job role.

**Request** вЂ” `multipart/form-data`

| Field     | Type   | Description                  |
|-----------|--------|------------------------------|
| `resume`  | File   | PDF file of the resume       |
| `jobRole` | String | Target job role (e.g. "Frontend Developer") |

**Response**

```json
{
  "success": true,
  "analysis": {
    "overall_score": 72,
    "top_strengths": ["..."],
    "critical_gaps": ["..."],
    "keywords_to_add": ["..."],
    "sections": {
      "technical_skills": { "score": 80, "summary": "...", "strengths": [], "suggestions": [], "suggested_rewrites": [] },
      "projects":         { "score": 65, "summary": "...", "strengths": [], "suggestions": [], "suggested_rewrites": [] },
      "experience":       { "score": 70, "summary": "...", "strengths": [], "suggestions": [], "suggested_rewrites": [] },
      "education":        { "score": 90, "summary": "...", "strengths": [], "suggestions": [], "suggested_rewrites": [] }
    }
  }
}
```

---

### `POST /api/email/send`

Generate an AI-written outreach email and send it via Brevo with the resume attached.

**Request** вЂ” `multipart/form-data`

| Field           | Type   | Description                     |
|-----------------|--------|---------------------------------|
| `resume`        | File   | PDF file of the resume          |
| `to`            | String | Recipient email address         |
| `jobRole`       | String | Target job role                 |
| `applicantName` | String | Applicant's full name           |

**Response**

```json
{
  "success": true,
  "email": {
    "subject": "...",
    "body": "..."
  },
  "meta": {
    "sentTo": "hr@company.com",
    "applicant": "Rishi Sharma"
  }
}
```

## рџ”„ Frontend Page Flow

```
Home
 в””в”Ђв–є Analysis Page   (POST /api/resume/analyse)
      в””в”Ђв–є Email Preview Page   (POST /api/email/send)
           в””в”Ђв–є Success Page

Data flows between pages via React Router v6 `useLocation` state:

```js
navigate('/analysis', {
  state: { analysis, jobRole, applicantName }
})
```

> **Note:** `File` objects cannot be serialised through Router state. The resume file is re-selected on the Analysis page before sending the email.

## рџљў Deployment (Render)

1. Push your backend to a GitHub repository.
2. Create a new **Web Service** on [Render](https://render.com).
3. Set the **Build Command** to `npm install` and **Start Command** to `node src/app.js`.
4. Add all environment variables in the Render dashboard under **Environment**.
5. Deploy вЂ” Render will auto-deploy on every push to `main`.

> **Important:** Render's free tier **blocks SMTP ports** (25, 465, 587). This is why the project uses Brevo's HTTP REST API instead of Nodemailer.

---

## рџ§  Key Technical Decisions

| Problem | Decision |
|---|---|
| Render blocks SMTP | Switched from Nodemailer в†’ Brevo REST API over HTTPS |
| `pdf-parse` ES Module errors | Switched to `pdf2json` with a `safeDecode` wrapper |
| Groq JSON parsing issues | Extract JSON with `/\{[\s\S]*\}/` regex, escape raw newlines before `JSON.parse` |
| File not serialisable via Router state | User re-selects the file on Analysis page instead of passing it through state |
| Context API overhead | Unnecessary for linear flow вЂ” `useLocation` state is sufficient |
